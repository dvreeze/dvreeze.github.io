Yaidom and namespaces

Yaidom is a uniform XML query API, written in Scala. Moreover, yaidom provides several specific-purpose DOM-like tree implementations adhering to this XML query API.

This article treats namespaces in yaidom.

Introduction

This article is the second one in a series of articles introducing the yaidom library. This article treats namespaces in yaidom. Like in the first article, basic XML querying in yaidom is shown, but this time paying special attention to qualified names, expanded names, namespace declarations and in-scope namespaces. It is shown that yaidom is an API that values precision, clarity and minimality (not to be confused with conciseness).

This article is to a large part based on the article Understanding XML Namespaces, which in my view offers an excellent explanation of XML namespaces. Much of the message of that article can be expressed in yaidom. This 2nd yaidom article shows that.

It is assumed that the reader has first read the first article about yaidom, which introduced the basics of element querying in yaidom. Like in the preceding article on yaidom, it is assumed that the reader knows the basics of XML (including namespaces), Java XML processing, and Scala with its Collections API.

The remainder of this article is organized as follows:

  1. Qualified and expanded names.
  2. Namespace declarations and in-scope namespaces.
  3. Equivalent XML documents.
  4. Conclusion (of this second article about yaidom).

Qualified and expanded names

Like in the preceding article on yaidom, all code examples below can be tried out in the Scala REPL (if the parent directory path of the example files is replaced). Scala 2.10.X is required, and yaidom 0.8.0 (or later) must be on the classpath. Before showing some basic yaidom element queries, assume the following code to have executed:

import java.io.File
import javax.xml.parsers._
import scala.collection.immutable
import eu.cdevreeze.yaidom._

// Using a yaidom DocumentParser that used SAX internally
val docParser = parse.DocumentParserUsingSax.newInstance

// Replace the following path!
val parentDir = new File("/path/to/parentdir")

val doc: Document =
    docParser.parse(new File(parentDir, "feed1.xml"))

val docElem = doc.documentElement
