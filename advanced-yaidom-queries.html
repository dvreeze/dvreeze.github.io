Introduction

This article is the second in a series of articles introducing a Scala DOM-like XML library, called yaidom ("yet another immutable DOM"). It is assumed that the reader has first read the first article about yaidom, which introduced the basics of element querying in yaidom. Like in the preceding article on yaidom, it is assumed that the reader knows the basics of XML (including namespaces), Java XML processing, and Scala with its Collections API.

This article treats more advanced element querying in yaidom. The remainder of this article is organized as follows:

  1. Introductory example.
  2. Elems versus ElemBuilders.
  3. Paths, and path-aware querying.
  4. More interesting example queries.
  5. Summary of element implementations and their query APIs.
  6. Conclusion (of this second article about yaidom).

The queries again use the same sample XML as in the first yaidom article.
